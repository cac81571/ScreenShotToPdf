# マイグレーション スクリーンショット → PDF

システムマイグレーションのスクリーンショットを、**画像フォルダA**（移行前）と**画像フォルダB**（移行後）で比較しやすい1本のPDFにまとめるツールです。加えて、**ピクセル差分を赤の半透明マスクで可視化**した結果を、別のPDFにまとめることもできます。

- **比較用PDF（PDF作成）**
  - **奇数ページ**: フォルダAの画像（ファイル名でソート）
  - **偶数ページ**: フォルダBの画像（ファイル名でソート）
- **差分PDF（差分PDF作成）**
  - リストの**全行**について、同じインデックスのA/B画像を比較し、差分マスクを重ねた画像を**交互ページ**で出力（通常の比較PDFと同じ並び）
  - 中間のPNGは `diff_overlay` フォルダに保存され、PDFはその親フォルダに `＜フォルダA名＞_diff_overlay.pdf` で保存される
- 対応画像: PNG, JPG, JPEG, GIF, BMP, WebP

## 技術要素

- **Java 11** + **Swing**（GUI）
- **Maven**（ビルド）
- **OpenPDF**（PDF生成）
- **FlatLaf**（Look and Feel）

## 必要環境

- Java 11 以上
- Maven 3.6 以上（ビルド時）

## ビルド・実行

```bash
# ビルド（依存関係をすべて含む起動可能 JAR が target に生成される）
mvn package -DskipTests

# 実行（単体 JAR で起動）
java -jar target/migration-screenshot-pdf-1.0.0-SNAPSHOT.jar
```

開発時は `mvn exec:java` でも起動できます。

## 使い方

1. 起動すると Swing のウィンドウが開きます。
2. **画像フォルダA** … 移行前のスクリーンショットが入ったフォルダのパスをコンボボックスに入力または選択する。
3. **画像フォルダB** … 移行後のスクリーンショットが入ったフォルダのパスをコンボボックスに入力または選択する。
4. **ファイル抽出** ボタンを押すと、両フォルダから画像ファイル（上記拡張子）を検出し、それぞれのリストに表示される。
5. リストでは次の操作ができる。
   - **選択削除** … 選択した行を削除する。
   - **前に空白追加** / **後ろに空白追加** … 選択行の前または後ろに「空白ページ」を1件追加する（PDFではラベルのみのページになる）。
6. **PDF作成** ボタンを押すと、フォルダAの**親ディレクトリ**に「フォルダA名.pdf」で PDF が保存され、そのフォルダがエクスプローラーで開く。
7. **差分PDF作成** ボタンを押すと、リスト全件について差分処理を行い、親ディレクトリに `フォルダA名_diff_overlay.pdf` を出力する。処理中に生成したPNGは親直下の `diff_overlay` フォルダに保存される。

同一インデックスの画像が、奇数ページ（A）と偶数ページ（B）で並ぶため、見比べながら確認しやすくなります。フォルダパスは履歴として `~/.screenShotToPdf/` に保存されます。

### 差分の調整（画面下部）

| 項目 | 説明 |
|------|------|
| **差分しきい値** | R/G/B の差の合計がこの値を超えたピクセルを差分とみなす（整数、0以上）。大きいほどノイズを無視しやすい。 |
| **赤α** | 差分部分に重ねる赤の不透明度（0.0〜1.0）。 |
| **マスク拡張** | 差分ピクセルを周囲へ広げる半径（px、0以上）。大きいほどマスクが太く見える。初期値は 10。 |
| **Aにマスク** / **Bにマスク** | それぞれの出力画像にマスクを重ねるか。既定は **Aはオフ、Bはオン**。 |

解像度がAとBで異なる場合は、**画像Aの幅・高さに合わせて画像Bをリサイズ**してから比較します。

差分PDFでは、画像が片方だけ欠ける行（空白ページやファイルなし）はスキップされ、その位置はPDF上では空白ページになります。

**日本語のファイル名・空白ページについて**: PDF 内のテキストは日本語表示に対応しています。次のいずれかの方法で日本語フォント（TTF）を用意してください。

- **推奨（Windows で確実に表示される方法）**  
  1. [IPAex フォント](https://ipafont.ipa.go.jp/) など日本語対応の **.ttf** をダウンロードする。  
  2. ユーザーホームの `.screenShotToPdf/fonts/` フォルダを作成し、その中に **.ttf** を 1 つ配置する（例: `C:\Users\<ユーザー名>\.screenShotToPdf\fonts\ipaexg.ttf`）。  
  3. アプリを起動し、PDF を作成する。

- 開発時は `src/main/resources/fonts/` に **.ttf** を置くと JAR に含まれます。  
- Windows の Meiryo 等（.ttc）は OpenPDF が扱えない場合があり、その場合は上記の TTF 配置が必要です。
