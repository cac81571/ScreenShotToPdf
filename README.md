# マイグレーション スクリーンショット → PDF

システムマイグレーションのスクリーンショットを、**移行前**と**移行後**で比較しやすい1本のPDFにまとめるツールです。

- **奇数ページ**: 移行前フォルダの画像（ファイル名でソート）
- **偶数ページ**: 移行後フォルダの画像（ファイル名でソート）
- 対応画像: PNG, JPG, JPEG, GIF, BMP, WebP

## 技術要素

- **Java 11** + **Swing**（GUI）
- **Maven**（ビルド）
- **OpenPDF**（PDF生成）
- **FlatLaf**（Look and Feel）

## 必要環境

- Java 11 以上
- Maven 3.6 以上（ビルド時）

## ビルド・実行

```bash
# ビルド（依存関係をすべて含む起動可能 JAR が target に生成される）
mvn package -DskipTests

# 実行（単体 JAR で起動）
java -jar target/migration-screenshot-pdf-1.0.0-SNAPSHOT.jar
```

開発時は `mvn exec:java` でも起動できます。

## 使い方

1. 起動すると Swing のウィンドウが開きます。
2. **移行前フォルダ** … 移行前のスクリーンショットが入ったフォルダのパスをコンボボックスに入力または選択する。
3. **移行後フォルダ** … 移行後のスクリーンショットが入ったフォルダのパスをコンボボックスに入力または選択する。
4. **ファイル抽出** ボタンを押すと、両フォルダから画像ファイル（上記拡張子）を検出し、それぞれのリストに表示される。
5. リストでは次の操作ができる。
   - **選択削除** … 選択した行を削除する。
   - **前に空白追加** / **後ろに空白追加** … 選択行の前または後ろに「空白ページ」を1件追加する（PDFではラベルのみのページになる）。
6. **PDF作成** ボタンを押すと、移行前フォルダの**親ディレクトリ**に「移行前フォルダ名.pdf」で PDF が保存され、そのフォルダがエクスプローラーで開く。

同一インデックスの画像が、奇数ページ（移行前）と偶数ページ（移行後）で並ぶため、見比べながら確認しやすくなります。フォルダパスは履歴として `~/.screenShotToPdf/` に保存されます。
